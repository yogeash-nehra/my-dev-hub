## Tooling Updates

### Ollama v0.23.0 — `ollama launch claude-desktop`
**Source:** ollama/ollama GitHub | **Date:** 2026-05-03 | **Link:** https://github.com/ollama/ollama/releases/tag/v0.23.0
**TL;DR:** Ollama now launches Claude Desktop directly — `ollama launch claude-desktop` installs the app and wires local Ollama models into Claude Cowork and Claude Code inside the desktop environment.
**Dev signal:** If you run local models via Ollama and also use Claude Desktop, this removes the terminal-only constraint. Note: Web Search and Extensions are unsupported in this integration (marked "coming soon"). Model recommendations are now server-driven, so featured-model lists update without a client upgrade. Also in this build: Metal initialization now fails gracefully on macOS when GGML kernel compilation fails rather than crashing.
**Primary source:** https://github.com/ollama/ollama/releases/tag/v0.23.0

### llama.cpp b9014 — WebGPU backend gains layer norm ops
**Source:** ggml-org/llama.cpp | **Date:** 2026-05-04 | **Link:** https://github.com/ggml-org/llama.cpp/releases
**TL;DR:** Layer normalization operations added to the WebGPU compute shader implementation (PR #22406) with improved floating-point stability versus the previous fallback path.
**Dev signal:** If you're serving or experimenting with llama.cpp in browser/WebGPU contexts, this closes an ops gap that forced CPU fallbacks during norm layers. Also shipped today: b9015 removes dead Vulkan code (PR #22621) and b9016 updates speculative decoding parameter docs to match the recent `--spec-draft-n-max/min` rename (PR #22397).
**Primary source:** https://github.com/ggml-org/llama.cpp/pull/22406
